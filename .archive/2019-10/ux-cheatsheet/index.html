Charlie's UX Cheatsheet

This is a page that has a collection of industry standards best practices, which I have personally revised, edited, and created. Using this as a single source of truth ensures consistency, and a basic standard.

Accessibility

Web

This is a big topic — here's a short list that are the most important:

  • Use semantic tags (e.g., h1 for headers)
  • Avoid using placeholders
  • Show text with icons, or text only; reserve independent icon use for home, logos, or ubiquitous imagery (e.g. Save, Exit, etc.)
  • Always show labels
  • Ensure color contrast is both pleasant, and strong enough

Copy

Use of "We" vs "You"

Use "you" most of the time: it focuses on the user and is stronger. Use "we" sparingly, e.g.: "We promise...", or when you have a close or intimate relation with the user.

Passive vs Active Voice

Use passive voice for web headings. Use active voice for everything else.

Forms

Layout

Use a vertical single column for most cases.

General

Don’t hide basic helper text (e.g., always show password requirements)

Confirmation

Don't have redundant confirmation fields (e.g., an email field and then a confirm-email field. You do need to make sure your Reset Password function work properly though!

Password

Have a toggle mask option. Use condensed text if possible instead of an icon.

Validation & Errors

Use inline validation BUT don't validate as a user is typing, validate only when the field loses focus.

General Design

Progress Bars

A completion meter can increase user satisfaction and engagement. Showing it can make long load times seem shorter. Use an indeterminate load bar if you don't know how long something will take.

Related: Zeigarnik effect.

Zeigarnik Effect

People remember incomplete or interrupted tasks better than completed tasks. People will usually want to complete a task (to effectively forget about them).

Related: Progress Bar.

Animation

CSS/HTML Optimization

Get 60fps! Do it properly. Here's a summary:

  • Don't use transitions with left/top/right/bottom properties, use transform
  • Include will-change: transform;
  • Isolate animating elements so it doesn't have any siblings

Easing

Make sure to use the correct easing. Usually use ease-out-cubic for intros, and ease-out-cubic for outros.
