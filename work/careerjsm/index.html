Helping job-seekers manage their job search

Job seekers have a lot to worry about. Sometimes, they've been out of the job hunt for a while and need to refresh their skills. If they've just lost a job, they can be in a sensitive emotional state. CareerJSM provides software to these people, and I ensure that by providing each user a tailored and comfortable experience, I can increase both sign-ups and user engagement.

When designing the onboarding/registration workflow, I used thoughtful copy, layout, and animation, to ease the user into the product with as little friction as possible. The resulting update increased user sign-up by close to 100%.

A Sample Job Tracker Page

Meeting user & business needs

To address our user's needs, I conducted user interviews and surveys to find areas of improvement. In one case, I discovered that many users have difficulty building authentic relationships with people (i.e., not a transactional relationship), so I facilitated a team design sprint to address these issues with a networking dashboard.

Through my time here, I've also worked on implementing a resume builder, a job tracker, a learning center, and an admin dashboard to manage it all.

An iteration of a user flow for signing up.

Measuring impact

It's not easy to determine how well new features are working, so I make sure to record, gather, and analyze data to evaluate target KPIs.

Some things I looked for were user satisfaction, ease-of-use, and overall value. I combined qualitative in-person testing with quantitative survey feedback to gather as much data as possible. Afterwards, I analyze the data, present findings, and make recommendations on how to proceed.

A blank networking dashboard that a user might see.

Quality assurance

To ensure a smooth and stress-free app, I wrote and ran integration tests to ensure features were continuously working. I also manually audited the app for responsive design, web accessibility (WCAG 2.0), and localization compatibility.
